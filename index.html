HTML

  • HTML is used to create the structure of a webpage.
  • HTML is made up of elements including: Text, Image, Header, and List elements.
  • Element types to divide and describe a page are :Meta-related elements, Content division elements, and Semantic elements.
  • The head element contains information about the webpage.
  • Meta elements contain information about the page that is used by the browser.
  • The body element represents the visible content shown to the user.
  • HTML elements are defined in the beginning element, never the closing.
  • Elements nested inside other elements are called child elements.
  • Semantic HTML elements the name describes its content and the role it plays on the webpage.
  • An element refers to an entire element from opening to closing tag, but a tag is only what's inside the brackets.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Internal style sheet contains CSS rules for the webpage in the header section of the HTML file.
  • The rules only apply to the page they're on.
  • Declarations contain the CSS property we want to apply and the value of the property.
  • Margin property acts like a margin in a word processing document.
  • Padding adds space around the content inside the element
  • Can combine two selectors into a single CSS rule with a comma, to allow it to affect both selectors.
  • DRY, or do not repeat yourself, is a principle that reuses or shares pieces of code to reduce the number of lines of code.
  • To view CSS changes, must link the CSS file to the HTML file.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout main can get us to the main branch if we are not already.
  • git pull origin main pulls down the latest main branch.
  • git add -A commits all files in the working branch.
  • git commit -m flags our commit as having a message which should be short and descriptive of the changes

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script
  • We use = to assign the specific data we want stored in our variable.
  • The console.log was created to make it easier for developers to test their code.
  • An "if" statement introduces decision making into our code.
  • A value is considered falsy if it is 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false
  • All other values are considered truthy.
  • === is an operator that checks to see if two values are equal.
  • An array is a group of related data in a variable./
  • Arrays are useful for sharing related items with diefferent data types, but not all arrays have multiple data types.
  • Index is a location of the data item in an array.
  • The first item of an index is given the value of 0.
  • A for loop uses a pattern of indices to perform a task on all items in an array by executing a single block over and over.
  • A for loop contains the starting point of the loop, a condition, and the final statement.
